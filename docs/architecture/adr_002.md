# ADR-002: Implementacja silnika synchronizacji czasowej Timera (Unix Delta Timestamp)

## Status
**ACCEPTED**

## Data
2026-05-18

## Kontekst

Progresywne Web Aplikacje (PWA) działające w przeglądarkach mobilnych napotykają poważny problem z **background throttling**:

- **Google Chrome** - Throttle'uje `setInterval` do 1s gdy tab jest w tle
- **Safari** - Usypia karty zupełnie po kilku sekundach w tle
- **Klasyczny timer** (`setInterval(() => seconds--, 1000)`) **driftuje** o kilkanaście minut przy dłuższych sesjach focus

### Problem biznesowy:

Użytkownik z ADHD uruchamia 25-minutową sesję focus, minimalizuje przeglądarkę by nie rozpraszać się internetem. Po 30 minutach wraca - timer pokazuje **wciąż 5 minut do końca** (drift 10+ minut). Użytkownik traci zaufanie do systemu.

### Wymagania:

1. **Precyzja** - Błąd maksymalny ≤ 1 sekunda niezależnie od stanu karty
2. **Background resilience** - Timer działa poprawnie przy zminimalizowanej przeglądarce
3. **Battery friendly** - Nie obudzaj procesora niepotrzebnie
4. **Global state** - Timer widoczny we wszystkich komponentach aplikacji

## Decyzja

Implementacja **Unix Delta Timestamp** w globalnym `TimerContext`:

```typescript
// Architektura Unix Delta
const TimerContext = createContext<TimerContextValue | null>(null);

export const TimerProvider = ({ children }) => {
  const [timeLeft, setTimeLeft] = useState(0);
  const [timerState, setTimerState] = useState<'idle' | 'running' | 'paused'>('idle');
  const expectedEndTimeRef = useRef<number>(0);
  
  const startTimer = (minutes: number) => {
    const durationMs = minutes * 60 * 1000;
    expectedEndTimeRef.current = Date.now() + durationMs;
    setTimerState('running');
    
    // Interval throttled - ale Date.now() zawsze aktualny
    intervalRef.current = setInterval(() => {
      const now = Date.now();
      const remaining = Math.max(0, expectedEndTimeRef.current - now);
      setTimeLeft(remaining);
      
      if (remaining === 0) {
        completeTimer();
      }
    }, 1000);
  };
  
  return (
    <TimerContext.Provider value={{ timeLeft, timerState, startTimer }}>
      {children}
    </TimerContext.Provider>
  );
};
```

### Kluczowe cechy:

1. **Immutable expectedEndTime** - Ustalony przy starcie, nie zmienia się
2. **Dynamic delta calculation** - `remaining = expectedEnd - Date.now()`
3. **Throttling immune** - Nawet jeśli interval jest throttle'owany do 1s, delta jest prawidłowa
4. **Global singleton** - Stan poza cyklem życia komponentów, brak race conditions

## Konsekwencje

### Pozytywne:
- ✅ **Timer odporny na throttling** - Działa poprawnie przy zminimalizowanej karcie
- ✅ **Precyzja** - Błąd max 1s (granularność interval), nie kumuluje się
- ✅ **Background resilient** - Przeżycie usypiania kart przez Safari
- ✅ **Global state** - Dostęp z każdego ekranu, persystencja przy nawigacji
- ✅ **Battery efficient** - Tylko jeden interval, brak ciągłych obliczeń

### Negatywne:
- ❌ **Granularność 1s** - Nie nadaje się do milisekundowego odliczania (akceptowalne dla Pomodoro)
- ❌ **Brak sync systemowego** - Nie korzysta z systemowego alarm clock (celowo - brak notyfikacji push)

## Porównanie z alternatywami

| Podejście | Problem |
|-----------|---------|
| Klasyczny `seconds--` | Driftuje przy throttlingu |
| `requestAnimationFrame` | Zatrzymuje się w tle |
| `Web Workers` | Overkill, złożoność komunikacji |
| `Service Worker + Push` | Wymaga uprawnień, złożona konfiguracja |

## Implementacja referencyjna

```typescript
// hooks/useTimer.ts - Przykład użycia
const useTimer = () => {
  const context = useContext(TimerContext);
  if (!context) throw new Error('useTimer must be used within TimerProvider');
  
  const { timeLeft, startTimer, pauseTimer } = context;
  
  // Formatowanie dla UI
  const minutes = Math.floor(timeLeft / 60000);
  const seconds = Math.floor((timeLeft % 60000) / 1000);
  
  return { minutes, seconds, startTimer, pauseTimer };
};
```
