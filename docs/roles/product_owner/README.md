# Rola: Product Owner (PO)

> **Odpowiedzialność:** Roadmapa, priorytetyzacja zadań i zarządzanie Backlogiem.

---

## Zakres Odpowiedzialności

### 1. Roadmap Management
- Utrzymywanie aktualnej Roadmapy w `docs/business/roadmap.md`
- Planowanie faz i deliverables
- Koordynacja między zespołami (UX, Dev, QA)

### 2. Backlog Prioritization
- Priorytetyzacja zadań według MOSCOW (Must/Should/Could/Won't)
- Definiowanie kryteriów akceptacji dla każdego zadania
- Zarządzanie technicznym długiem

### 3. Requirements Definition
- Tworzenie User Stories dla ADHD persona
- Definiowanie Acceptance Criteria
- Utrzymywanie zgodności z ICP (Ideal Customer Profile)

### 4. Release Planning
- Planowanie wersji i milników
- Go/No-Go decyzje na podstawie weryfikacji QA
- Release notes i komunikacja

## Deliverables

- Aktualizacje `roadmap.md` co sprint/tydzień
- Definicja zadań w backlogu (z AC)
- Decyzje Go/No-Go przed release

---

## ADHD-Focused Priorities

Priorytetyzacja musi uwzględniać ADHD persona:

1. **Friction Reduction** - minimalne tarcie w flow
2. **Visual Persistence** - zadania muszą być widoczne
3. **Decision Support** - system myśli za użytkownika
4. **480px Constraint** - mobile-first zawsze
